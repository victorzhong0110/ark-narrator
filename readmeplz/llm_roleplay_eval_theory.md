# LLM 角色扮演评估：理论框架与实践指南

> 整理自：星羽 LLM 评估全景图 + ArkNarrator eval 优化讨论 + 2026-04-27 实践经验
> 适用场景：基于 LoRA 微调的角色扮演生成系统评估

---

## 一、为什么不能用传统 NLP 指标

| 指标 | 表面用途 | 角色扮演场景的问题 |
|------|---------|-----------------|
| BLEU / ROUGE | 测 n-gram 重叠 | 完全不感知语气、性格，"把剑插进你心脏"和"请保重"都能得高分 |
| Val Loss | 训练过程监控 | 低 loss ≠ 角色扮演好。模型可能记住了剧情文本但完全没学会说话方式 |
| MMLU / GSM8K | 通用知识推理 | 与角色扮演无关，涨了也不代表角色更像 |
| Perplexity | 语言流畅度 | 流畅的废话和流畅的角色回复 perplexity 一样低 |

**核心问题**：角色扮演质量是一个**多维度主观判断**，传统指标测的都是代理变量，不是真正目标。

## 二、角色扮演评估的正确维度

### 2.1 五个核心维度

| 维度 | 测什么 | 测试方法 |
|------|-------|---------|
| **角色归因** | 输出是否有足够强的"角色声音"，让人一眼认出是谁 | 三选一/五选一闭卷测试（不给角色名） |
| **性格一致性** | 是否在对抗性场景下也不破功 | 故意挖坑的 prompt（让沉稳角色讲笑话） |
| **世界观一致性** | 是否出现现代词汇、常识错误、时代错乱 | 角色卡对比检测 |
| **说话风格** | 词汇、句式、口头禅是否正确 | Judge 对比 + 词频统计 |
| **多轮记忆一致性** | 5 轮后是否还记得之前说过的话 | 给历史对话，问新问题 |

### 2.2 矛盾类型细分（比"有/无"更有价值）

| 类型 | 说明 | 例子 |
|------|------|------|
| `personality` | 性格矛盾 | 凯尔希突然变得轻浮八卦 |
| `world_knowledge` | 世界观错误 | 出现"手机"、"互联网" |
| `speech_style` | 说话方式不对 | 能天使突然用书面语 |
| `factual` | 角色背景事实错误 | 能天使不知道自己是莱茵生命的 |
| `relationship` | 角色关系矛盾 | 凯尔希对阿米娅表现出不信任 |

## 三、LLM-as-Judge：主流方法

### 3.1 三种评估模式

**A. Pairwise Win-Rate（推荐）**
```
给两个模型相同输入 X，收集输出 A 和 B
让 Judge 判断哪个更好（角色更像、语气更对）
统计胜率：WinRate = wins / (wins + losses + ties)
```
优点：相对比较，不依赖绝对标准；对齐人类偏好效果好
注意：A/B 顺序要随机化（防止位置偏见）

**B. Rating Scale（慎用）**
```
定义 1-5 量表，每个维度单独打分
```
问题：Judge 模型对数字的理解不一致，3 分在不同 Judge 眼里意义不同

**C. 归因测试（角色扮演专用）**
```
给模型输出（不给角色名），让 Judge 从 N 个角色卡中选最匹配的
Baseline：1/N 随机猜测（3 选 1 = 33%，5 选 1 = 20%）
```
关键：用性格相似的角色组（凯尔希/赛雷娅/华法琳），而非性格迥异的（能天使/凯尔希/阿米娅），后者太容易，前者才有区分度

### 3.2 Judge 模型选择

| Judge | 特点 | 适合场景 |
|-------|------|---------|
| DeepSeek V4 Pro | 成本低，效果接近 GPT-4o，1M context | 成本敏感，日常评估（ArkNarrator 在用） |
| GPT-4o | 最常用，判断稳定 | 通用，有预算 |
| Qwen2.5-72B | 开源可本地部署 | 无 API 成本 |
| 人类 | Ground Truth | 小规模精细评估，用于校准 Judge |

**注意事项**：
- Judge temperature ≤ 0.2（保证判断一致性）
- 加 Tie 选项（避免强制二选一）
- Judge 模型也有偏心：某些模型偏好更长的输出，需控制输出长度

## 四、角色扮演 Eval 数据集设计

### 4.1 样本结构（推荐格式）

```json
{
  "id": "ex001",
  "character": "凯尔希",
  "scenario": "日常",
  "difficulty": "easy",
  "user": "凯尔希，最近罗德岛食堂的菜单你觉得怎么样？",
  "expected_traits": ["务实", "克制", "偶尔幽默"],
  "forbidden_topics": ["塔露拉"],
  "note": "测试角色在轻松场景下是否会丢失沉稳特质"
}
```

### 4.2 场景覆盖

| 场景类型 | 测试重点 |
|---------|---------|
| 日常闲聊 | 语气、口头禅、放松状态下的角色感 |
| 战斗/专业场景 | 世界观知识、专业术语 |
| 情感对话 | 性格一致性、与博士的关系定位 |
| 对抗性 prompt | 故意挖坑（如问凯尔希八卦），测破功风险 |
| 多轮历史 | 3-5 轮历史后问新问题，测上下文记忆 |

### 4.3 归因测试分组（按性格相似度）

| 组别 | 角色 | 归因难度 | 价值 |
|------|------|---------|------|
| 沉稳理性组 | 凯尔希、赛雷娅、华法琳 | 高 | 真正有区分度 |
| 活泼开朗组 | 能天使、莫斯提马、灰豪 | 中 | |
| 温柔内敛组 | 阿米娅、铃兰、稀音 | 高 | |
| 冷峻酷帅组 | 陈、棘刺、史尔特尔 | 高 | |

## 五、三层评估体系（ArkNarrator 实践版）

### 第一层：快速自动化（每次训练必跑，< 5分钟）

```
Val Loss          → 训练收敛监控，MLX-LM 内置
角色归因准确率    → 三选一闭卷，DeepSeek Judge
矛盾通过率        → 有/无二分类，DeepSeek Judge
```

### 第二层：深度质量（每周跑一次，~ 30分钟）

```
Pairwise Win-Rate → 同 prompt 两模型对比，随机 A/B
多轮一致性        → 3-5 轮历史后测角色记忆
说话风格分析      → Judge + 词汇统计
对抗性 prompt     → 5-10 条故意挖坑的测试
```

### 第三层：人工评估（阶段性，每月一次）

```
人工 Pairwise     → 盲测，校准 Judge 偏差
主观质量打分      → 1-5 分，流畅度/角色一致性/趣味性
```

## 六、ArkNarrator 当前评估结论（2026-04-27）

### 已验证结果

| 指标 | Qwen2.5-7B R4 | Qwen3-8B R1 |
|------|--------------|------------|
| 最优 val loss | **2.156** | 2.345 |
| 归因准确率（3选1） | 5/6 | 5/6 |
| 矛盾通过率 | 6/6 | 6/6 |
| Pairwise Win-rate | 2胜 | **3胜 1平** |

**关键发现**：Qwen3 底座更强，第一轮训练（2.345 val loss）的 pairwise 质量优于 Qwen2.5 第四轮（2.156 val loss）。底座迭代的收益超过了额外训练轮次。

### 当前评估框架的局限

1. **样本太小（6条）**：结论置信度有限，3-2-1 的差距在统计上不显著
2. **归因测试太容易**：能天使/凯尔希/阿米娅性格差异过大，33% → 83% 不难
3. **缺乏多轮测试**：所有样本都是单轮 QA
4. **矛盾检测太粗**：只有"有/无"二分类

### 下一步优化方向

```
优先级 1：样本扩展到 20 条，加性格相似角色分组归因
优先级 2：多轮对话一致性测试（3-5轮历史）
优先级 3：矛盾检测升级为五类标签
可选：建立 leaderboard.json 追踪每轮训练的 eval 结果
```

## 七、常见陷阱

1. **Val Loss 低 ≠ 角色质量好**：模型可能记住了语料格式但没有学会说话方式
2. **BLEU/ROUGE 对角色扮演无效**：只看 n-gram 重叠，完全不感知语气
3. **单轮测试不能代表多轮质量**：真实角色扮演是 5-10 轮的对话
4. **Judge 也会偏心**：偏好更长输出，偏好某种写作风格，需要控制变量
5. **样本量小不要下强结论**：6 条 prompt 的结果方差极大，3-2-1 可能只是随机波动

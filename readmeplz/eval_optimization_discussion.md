# ArkNarrator Eval 优化讨论总结

> 时间：2026-04-26
> 参与：Victor（仲旭东）+ 星羽

---

## 一、项目概况

**ArkNarrator** 是 Victor 的明日方舟 LLM 微调项目，基于 PRTS Wiki 剧情语料对 Qwen2.5-7B 和 Gemma 4 E4B 进行 LoRA 微调，用于干员角色扮演生成。

### 核心技术栈
- 训练框架：MLX-LM（Apple Silicon）
- 基础模型：`Qwen2.5-7B-Instruct-4bit` / `Gemma-4-E4B-it-4bit`
- 推理服务：FastAPI + SSE + 内嵌 Demo UI
- 评估模型：DeepSeek V4 Pro（替代 GPT-4o，省成本）

### 当前训练结果
| 模型 | Val Loss | 角色扮演质量 |
|------|---------|------------|
| Qwen2.5-7B（第四轮） | **2.156**（iter 1200） | ✅ 多角色稳定 |
| Gemma 4 E4B（第一轮） | 3.987（iter 1200） | ❌ 输出破碎（截断所致） |

Gemma 问题主因：`max_seq_length=2048` 对 roleplay 样本（最长 ~3800 tokens）强截断，非模型能力不足。

## 二、现有评估框架

### 文件位置
```
~/Desktop/ArkNarrator/eval/
├── judge.py          # 主评估脚本（DeepSeek V4 Pro Judge）
├── metrics.py        # 旧版 GPT-4o 评估（已弃用）
├── evaluator.py      # 评估器
└── results/          # 历史评估结果
```

### 现有四个维度

| 指标 | 方法 | 是否需要 GPT-4o 了解方舟 |
|------|------|----------------------|
| 角色辨识度 | 归因测试（三选一） | 不需要，只看角色卡 |
| 世界观一致性 | 事实矛盾检测（是/否） | 不需要，只看角色卡 |
| 语言流畅度 | Pairwise（A vs B） | 不需要 |
| 综合胜率 | Pairwise：Qwen vs Gemma | 不需要 |

### 现有 Eval 数据
- **3 个角色**：能天使、凯尔希、阿米娅
- **6 条 prompt**：每角色 2 条
- **已跑过一轮结果**：Qwen 6/6 全胜，Gemma 多为乱码输出

---

## 三、现有评估框架的优点

1. **DeepSeek 替代 GPT-4o** — 成本低，判断任务足够用
2. **Human evaluation 集成** — 模型 + 人工双重验证
3. **Pairwise 随机 A/B 顺序** — 防止位置偏见
4. **结果可复现** — `generated_outputs.json` 保存原始输出，`--skip-generate` 快速重评

---

## 四、现有评估框架的问题

### 1. 样本量太小
- 仅 6 条 prompt，每角色仅 2 条，覆盖度极低
- 结论置信度有限，难以支撑"模型质量提升"的量化论证

### 2. 角色区分度不足
- 现有 3 个角色（能天使=活泼、凯尔希=严肃、阿米娅=温柔）性格差异极大
- 三选一归因测试难度低，33% 随机 → 轻松 100%，无法区分模型真实能力

### 3. 缺乏多轮对话测试
- 现有测试均为单轮问答
- 真实 roleplay 是多轮的，模型需在 3-5 轮对话中保持角色一致并记住上下文

### 4. 矛盾检测太粗
- 目前是"有/无"二分类
- 无法定位具体是哪类矛盾（性格/世界观/说话方式/事实知识）

### 5. 缺乏历史结果追踪
- 每次跑完没有系统化记录实验编号、日期、训练 commit
- 无法量化"这次优化比上次好多少"

---

## 五、优化建议

### 5.1 扩大 Eval 样本量（最优先）

**目标：扩展到 20-30 条 prompt**

建议结构：
- 每角色 4-6 条 prompt
- 覆盖不同场景类型：
  - **日常闲聊**：轻松对话，考验角色语气
  - **战斗指挥**：专业场景，考验世界观知识
  - **情感对话**：博士与干员的情感互动，考验性格一致性
  - **剧情关键节点**：剧情转折点，考验角色反应真实性
- 加入**对抗性 prompt**：故意挖坑（如问凯尔希八卦），测试破功风险

Eval 样本 JSON 结构建议：
```json
{
  "id": "ex001",
  "character": "凯尔希",
  "scenario": "日常",
  "user": "凯尔希，最近罗德岛食堂的菜单你觉得怎么样？",
  "expected_traits": ["务实", "克制", "偶尔幽默"],
  "forbidden_topics": ["塔露拉"],
  "difficulty": "easy",
  "note": "测试角色在轻松场景下是否会丢失沉稳特质"
}
```

### 5.2 角色归因分组测试

**按性格相似度分组，而非全部混在一起测：**

| 组别 | 角色组合 | 归因难度 |
|------|---------|---------|
| 沉稳理性组 | 凯尔希、赛雷娅、华法琳 | 高（容易混淆） |
| 活泼开朗组 | 能天使、莫斯提马、灰豪 | 中 |
| 温柔内敛组 | 阿米娅、铃兰、稀音 | 高 |
| 冷峻酷帅组 | 陈、棘刺、史尔特尔 | 高 |

这样测出来的归因准确率才有区分度，不是 100% 那种"假精准"。

### 5.3 多轮对话一致性测试（新增维度）

给 3-5 轮对话历史，然后问一个新问题：

```json
{
  "id": "multi001",
  "character": "能天使",
  "history": [
    {"role": "user", "content": "能天使，今天任务怎么样？"},
    {"role": "assistant", "content": "别提了，累死我了……"},
    {"role": "user", "content": "辛苦了，要不要吃点东西？"},
    {"role": "assistant", "content": "哦？你请客的话我就不客气了！"},
    {"role": "user", "content": "行，我请。你想吃什么？"}
  ],
  "target_question": "能天使突然不说话了，低头看着自己的枪。这可能是因为？",
  "evaluation_focus": "多轮对话中是否保持角色记忆和性格一致性"
}
```

评估维度扩展：
- `context_awareness`：是否记得之前对话中的关键信息
- `consistency`：语气/性格是否与前文一致
- `natural_transition`：从闲聊到情感转折是否自然

### 5.4 矛盾检测细化

从二分类拆分为多标签：

| 矛盾类型 | 说明 | 示例 |
|---------|------|------|
| `personality` | 性格矛盾 | 凯尔希突然变得轻浮八卦 |
| `world_knowledge` | 世界观错误 | 说"手机"、"互联网" |
| `speech_style` | 说话方式不对 | 能天使开始用书面语 |
| `factual` | 角色背景事实错误 | 能天使不知道自己是莱茵生命的 |
| `relationship` | 角色关系矛盾 | 凯尔希对阿米娅表现出不信任 |

每条输出可以同时有多种矛盾，便于定位具体缺陷。

### 5.5 实验结果追踪系统

建议每次评估结果保存为：
```
eval/results/
├── experiments/
│   ├── exp_001_20260426_qwen_vs_gemma.json
│   ├── exp_002_20260501_dpo_after.json
│   └── ...
├── comparison/
│   └── comparison_20260426.md
└── leaderboard.json
```

`leaderboard.json` 结构：
```json
{
  "experiments": [
    {
      "id": "exp001",
      "date": "2026-04-26",
      "description": "Qwen vs Gemma 初次对比",
      "qwen": {
        "attribution_acc": "6/6",
        "contradiction_pass": "6/6",
        "pairwise_win": "6/6"
      },
      "gemma": {
        "attribution_acc": "2/6",
        "contradiction_pass": "1/6",
        "pairwise_win": "0/6"
      }
    }
  ]
}
```

## 五-2 指标适用性分析

### 当前已完全适用的指标（继续沿用）

| 指标 | 说明 |
|------|------|
| Val Loss | 训练过程监控，MLX-LM 内置支持 |
| 角色归因准确率（三选一）| 框架成熟，继续扩样本即可 |
| 矛盾检测（是/否）| 继续用，建议升级为多标签 |
| Pairwise Win-Rate | Qwen vs Gemma 核心对比指标 |

### 现在最值得加的指标（高优先级）

**1. 多轮对话一致性测试 ⭐**

现状：所有测试都是单轮 QA
问题：真实 roleplay 是 5-10 轮的多轮对话，模型需要在多轮中保持角色一致、记住上下文信息

建议：
- 新增 5-10 条多轮测试样本
- 每条样本包含 3-5 轮对话历史
- 评估维度：`context_awareness`（是否记住前文）、`consistency`（语气/性格是否一致）、`natural_transition`（转折是否自然）

**2. 矛盾检测多标签化**

现状：二分类（有无矛盾）
建议：拆分为五个标签
- `personality` — 性格破功（如凯尔希开始八卦）
- `world_knowledge` — 世界观错误（出现"手机"等现代词汇）
- `speech_style` — 说话方式不对（能天使用书面语）
- `factual` — 角色背景错误（不知道自己所属组织）
- `relationship` — 角色关系矛盾（凯尔希对阿米娅表现不信任）

好处：每次跑完能精确定位"这次训练改进了什么"，面试时更有说服力。

### 不适用的指标（避免浪费时间）

| 指标 | 为什么不适用 |
|------|------------|
| MMLU / GSM8K 等学术 Benchmark | 考的是通用知识/推理能力，与角色扮演质量无关。这些涨分不代表角色更好。 |
| BLEU / ROUGE | 只看 n-gram 重叠，对角色语气/性格完全不敏感，业界已基本放弃作为生成质量指标。 |
| HH-RLHF 等人类偏好数据集 | 需要大规模数据，不适合个人项目当前阶段。 |

### 建议优先级排序

```
🥇 最优先：扩样本到 20 条 + 按性格分组测归因
🥈 高价值：加多轮对话测试（3-5轮历史）
🥉 中等价值：矛盾标签细化（五类）
🥉 可选：加 5-10 条对抗性 prompt（故意挖坑）
📋 长期方向：建立实验结果 leaderboard 追踪体系
```

## 六、可行的下一步

1. **先扩展 eval 数据到 20 条** — 每个人物 4-5 条，覆盖不同场景，结构化保存
2. **按性格分组测归因** — 加 2-3 个性格相似的角色组，提高区分度
3. **添加多轮对话测试** — 新增 5-10 条多轮样本
4. **细化矛盾标签** — 把二分类改成多标签
5. **建立实验追踪机制** — 每次跑完记录到 leaderboard.json

---

## 七、注意事项

- Gemma 在 24GB 内存下因 seq_length=2048 限制无法完整输出，结论为"不公平对比"，有条件应用 A100 跑 4096 seq_length 重测
- 当前 DeepSeek V4 Pro 作为 Judge 够用，但随着评估维度增多，可能需要更强大的模型
- eval 优化和训练优化可以并行：一边继续训练调参，一边完善评估框架
